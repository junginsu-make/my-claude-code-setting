# 작업 이어하기 시스템

세션 간 작업 연속성을 보장하는 시스템입니다.

## 진행상황 저장 (/save-progress)

현재 작업 상태를 프로젝트의 `.claude/WORK_IN_PROGRESS.md`에 저장합니다.

### 저장 형식

```markdown
---
saved_at: [ISO 날짜시간]
session_id: [세션 ID]
project_dir: [프로젝트 경로]
---

# Work In Progress

## 현재 작업
[지금 하고 있는 작업의 핵심 내용]

## 완료된 항목
- [x] 완료1
- [x] 완료2

## 진행 중인 항목
- [ ] 진행중1 (상태: ~60%)
- [ ] 진행중2 (상태: 시작됨)

## 다음 할 일
1. [다음에 해야 할 것]
2. [그 다음에 해야 할 것]

## 핵심 컨텍스트
[이어서 작업할 때 반드시 알아야 할 정보]
- 변경된 파일들
- 주요 결정사항
- 주의사항/블로커

## 관련 파일
- [수정한 주요 파일 경로들]
```

### 저장 규칙

1. 프로젝트 루트를 감지합니다 (git root 또는 현재 디렉토리)
2. `.claude/` 폴더가 없으면 생성합니다
3. 기존 `WORK_IN_PROGRESS.md`가 있으면 덮어씁니다
4. 저장 후 사용자에게 "진행상황이 저장되었습니다" 안내합니다

## 작업 복원 (/resume-work)

`.claude/WORK_IN_PROGRESS.md`를 읽어서 이전 작업 맥락을 복원합니다.

### 복원 절차

1. 현재 프로젝트의 `.claude/WORK_IN_PROGRESS.md`를 읽습니다
2. 파일이 없으면 "이전 작업 기록이 없습니다"라고 안내합니다
3. 파일이 있으면:
   - 저장 시점과 경과 시간을 표시합니다
   - 완료/진행중/다음할일을 요약합니다
   - "이전 작업을 이어서 진행하시겠습니까?" 질문합니다
   - 사용자가 동의하면 진행 중이던 항목부터 시작합니다
4. 작업 시작 시 `WORK_IN_PROGRESS.md`의 상태를 "resumed"로 업데이트합니다

## 자동 동작

### 세션 시작 시 (SessionStart 훅)
- 프로젝트에 `.claude/WORK_IN_PROGRESS.md`가 존재하면 자동 감지
- 훅이 메시지를 출력: "이전 작업 기록이 있습니다. /resume-work 로 이어서 할 수 있습니다"

### 세션 종료 시 (Stop 훅)
- 도구 호출이 10회 이상이었으면 자동으로 진행상황 저장 제안
- 메시지: "진행상황을 저장하시겠습니까? /save-progress"
