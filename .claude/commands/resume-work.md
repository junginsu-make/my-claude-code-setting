# Resume Work

이전 세션의 작업을 이어서 진행합니다.

## 수행할 작업

1. 현재 프로젝트 루트에서 `.claude/WORK_IN_PROGRESS.md` 파일을 찾습니다
2. 파일이 없으면: "이전 작업 기록이 없습니다. 새로운 작업을 시작하세요." 안내
3. 파일이 있으면:
   a. 파일 내용을 읽습니다
   b. `saved_at` 날짜를 확인하여 경과 시간을 계산합니다
   c. 사용자에게 요약을 보여줍니다:
      - 마지막 저장 시점 (예: "2시간 전")
      - 완료된 항목 수 / 진행 중 항목 수
      - 다음 할 일 목록
   d. "이전 작업을 이어서 진행하시겠습니까?" 질문합니다
   e. 사용자가 동의하면:
      - 핵심 컨텍스트와 관련 파일을 바탕으로 현재 상태를 파악합니다
      - 진행 중이던 항목의 첫 번째부터 이어서 시작합니다
      - WORK_IN_PROGRESS.md의 status를 "resumed"로 업데이트합니다

## 표시 형식

```
╔══════════════════════════════════════════╗
║  이전 작업 기록 발견                       ║
╠══════════════════════════════════════════╣
║  저장 시점: 2시간 전 (2026-03-17 19:52)  ║
║  완료: 3개 / 진행중: 2개 / 다음: 4개      ║
╚══════════════════════════════════════════╝

## 현재 작업
[작업 내용 요약]

## 진행 중인 항목
- [ ] 항목1 (~60%)
- [ ] 항목2 (시작됨)

## 다음 할 일
1. 다음1
2. 다음2

이전 작업을 이어서 진행하시겠습니까?
```

## 규칙
- 관련 파일 목록의 파일들이 실제로 존재하는지 확인합니다
- 존재하지 않는 파일이 있으면 사용자에게 알립니다
- git status를 확인하여 저장 이후 변경된 사항이 있는지 체크합니다
- 복원 후 바로 작업을 시작합니다
